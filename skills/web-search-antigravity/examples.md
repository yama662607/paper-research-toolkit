# Web Search - Examples

このドキュメントでは、Web 検索スキルの具体的な使用例を示します。

※ 簡単なブラウジングや単発の確認は、CodexのデフォルトWeb Search toolを優先してください。本スキルは深掘り調査向けです。

## 例 1: 技術情報の検索

### シナリオ

Next.js 15 の新機能について調べたい

### 手順

```bash
bash ~/.paper-tools/skills/web-search-antigravity/scripts/web-search.sh "目的: Next.js 15の新機能の全体像を把握したい。範囲: 公式リリースノートと公式ブログを優先。観点: 新機能一覧/破壊的変更/移行時の注意点。形式: 要約→機能一覧表→移行チェックリスト。"
```

### 期待される結果

1. Next.js 15 の公式リリースノートや技術記事を検索
2. 主要な新機能をリストアップ
3. 各機能の概要と使い方を説明

## 例 8: 複数クエリの並列調査（1回の実行で深掘り）

### シナリオ

OSSのセキュリティ対策（MCPのセキュリティベストプラクティス）を、一次情報・二次情報・最新更新の3軸で深掘りしたい

### 手順

```bash
bash ~/.paper-tools/skills/web-search-antigravity/scripts/web-search.sh -D deep \
  -q "目的: MCP(Model Context Protocol)のセキュリティベストプラクティスを一次情報中心に整理。範囲: 公式仕様/公式ガイド/一次発表。観点: プロンプトインジェクション対策/最小権限/監査ログ/サンドボックス。形式: 要約→観点別整理→追加調査の差分→出典一覧。" \
  -q "目的: MCPのセキュリティに関する二次情報（ブログ/解説/レビュー）を補助的に確認し、一次情報との差分や注意点を整理。観点: 誤解されやすいポイント/実装上の落とし穴。形式: 差分中心の整理。" \
  -q "目的: 直近の更新/変更点のみを抽出し、セキュリティ面で影響がある差分を一覧化。範囲: 直近6〜12ヶ月の更新情報。形式: 変更点→影響→参考URL。"
```

### 期待される結果

1. 公式一次情報に基づく整理
2. 二次情報での補助と差分説明
3. 最新更新の抽出と影響評価

## Deep Research のベストプラクティス

深掘り調査を安定させるために、以下の原則に沿って依頼文を作成してください。

1. **目的を明確化**: 何を意思決定したいか、どのレベルの確度が必要かを冒頭で明記する。
2. **調査範囲の限定**: 期間・地域・対象製品・バージョンなどのスコープを絞る。
3. **一次情報を優先**: 公式仕様・規格・法令・一次発表を優先し、二次情報は補助に限定する。
4. **観点の列挙**: 比較軸・確認事項・評価指標を箇条書きで指定する。
5. **出力形式の指定**: 要約→詳細→差分→出典のように構成を固定する。
6. **追加調査の方針**: 「不足点を列挙→追加検索→差分更新」を明記する。
7. **複数クエリ運用**: 1回の実行で `-q` を複数指定し、目的別に並列調査する。

### Deep Research の依頼文テンプレ

```
目的: 〜を判断/設計/導入するための根拠が欲しい
範囲: 期間/地域/製品/バージョンを明記
優先情報源: 公式仕様/法令/一次発表を優先
観点: 確認したい点を箇条書きで列挙
形式: 調査理由→要約→観点別整理→追加調査の差分→出典一覧
追加調査: 不足点を明記して最低1回の追加検索を実施
```

## 調査依頼書の書き方（テンプレ）

短いキーワード列ではなく、以下のような「調査依頼書」形式で書くと精度が上がります。

```
目的: 何を判断/解決したいか
背景: なぜ調査が必要か（任意）
範囲: 対象領域・期間・地域・製品・バージョン
優先情報源: 公式ドキュメント/リリースノート/学術/報道など
観点: 比較軸や確認したいポイント
形式: 出力フォーマット（要約、表、箇条書き、コード例など）
制約: 公開日、言語、信頼性の条件など
```

## 検索クエリのポイント

### 効果的な検索クエリの作り方

1. **明確な質問形式**

   - 「〜について教えてください」「〜を調べてください」など、明確な依頼として記述
   - 知りたい内容を具体的に指定

2. **情報源の指定**

   - 「公式ドキュメントから」「リリースノートから」など、参照してほしい情報源を明記
   - より正確な情報が必要な場合に有効

3. **回答形式の指定**

   - 「コード例を含めて」「表形式で」など、望む回答の形式を指定
   - 「主要な〜をリストアップ」など、まとめ方を指示

4. **時期や条件の指定**

   - 「2025 年の」「最新の」など、情報の鮮度を指定
   - 「初心者向けに」「詳しく」など、詳細度を調整

5. **比較や分析の指示**
   - 「比較して」「違いを」など、分析の観点を明確に
   - 「メリット・デメリット」「使い分け」など、求める分析内容を指定
