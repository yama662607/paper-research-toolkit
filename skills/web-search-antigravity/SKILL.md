# Web Search (Antigravity)

このスキルは、`agy` コマンドを使用して Web 検索を実行し、複数ラウンドの深掘り調査と出典整理を行うためのものです。
簡単なブラウジングや単発の確認は、CodexのデフォルトWeb Search toolを使い、深い調査が必要なときに本スキルを使ってください。

## Examples

- 単発深掘り: 「approval_policy の挙動を公式ドキュメント中心に整理して」を 1 query で `agy` に投げ、出典付き report を得る。
- 複数 query 並列: `-q` を複数 + `-D deep` で目的別に並走し、全完了後にまとめて出力する。
- background 実行: 長時間 query は background terminal で走らせ、待ち中に並行調査を進める。

## Instructions

以下のコマンドを実行して、Web 検索を実行すること。
引数には検索したい内容や質問を自然言語で指定すること。

```
bash scripts/web-search.sh "<検索したい内容や質問>"
```

オプションを指定して検索品質や調査の深さ、タイムアウト挙動を調整すること。

```
bash scripts/web-search.sh -q "<検索内容>" -s example.com -r 365 -l ja
```

短いキーワード列ではなく、調査意図が伝わる文章で依頼すること。

```
bash scripts/web-search.sh -q "OpenAI Codex CLIのapproval_policy（on-request/untrusted/on-failure/never）の挙動の違いを公式ドキュメント中心に調べ、具体例付きで整理してください"
```

調査依頼書の形式（目的・範囲・観点・形式）で書くと精度が上がる。詳細は `examples.md` を参照すること。

大規模な調査を行う場合は `-D deep` を使うこと。

```
bash scripts/web-search.sh -q "<検索内容>" -D deep
```

時間効率のため、**1回の実行で複数の検索文を与えて調査**すること（`-q` を複数指定）。

```
bash scripts/web-search.sh -D deep \
  -q "目的: 公式一次情報（仕様/リリースノート）を中心に事実関係を整理して" \
  -q "目的: 主要な二次情報で補助し、差分や争点を確認して" \
  -q "目的: 最新の更新/変更点だけを抽出して"
```

複数クエリ実行時は、各クエリを独立タスクとして処理し、実行中は**クエリ完了の簡単な通知のみ**を表示し、**調査結果本体は全クエリ完了後にまとめて出力**される点に注意すること（結果の出力順はクエリ順）。

`-D deep` かつ複数クエリでは、**既定では安定性と速度のバランスを取って `-j 2` 相当**になる。さらに確実性を上げたいときだけ `-j 1` を明示指定すること。

実際に agy に送られるプロンプトを確認する場合は `-P` を使うこと。

```
bash scripts/web-search.sh -q "<検索内容>" -D deep -P
```

実行時間の計測が必要なら `-T` を使うこと。

```
bash scripts/web-search.sh -q "<検索内容>" -D deep -T
```

タイムアウトやリトライを調整する場合は以下を使うこと（デフォルトは 600 秒、deep は 3600 秒）。

```
bash scripts/web-search.sh -q "<検索内容>" -t 240 -R 2
```

`-D deep` の場合は、`-t` を指定しない限りタイムアウトを 3600 秒に自動設定する。`-t` を指定した場合はその値が最優先となり、デフォルト設定は上書きされる。

### 実行方式（background terminal）

- `web-search.sh` の実行は、原則として**background terminal** で行うこと
- 長時間実行中は、完了待ちで停止せず、**並行して独自調査**（論点整理、追加クエリ設計、一次情報候補の抽出、デフォルト検索での補完）を進めること
- Antigravity 実行結果の到着後に、並行調査で得た仮説・不足点を突合し、必要なら追加クエリを即時投入すること
- ユーザー向けには「実行開始」「並行調査中」「結果統合中」を短く進捗共有すること

### 出力の期待値

- 重要な主張には URL を添付
- 可能な限り公開日を明記（不明なら不明と明記）
- 出典一覧を最後にまとめる
  - `-f json` を指定した場合は JSON 形式で返る点に注意
- Antigravity CLI の起動時ログ（YOLO/credentials/Server...）はスクリプト側で除去される
- 進捗ログ（`[info ...]`）は既定でオフ。`--progress` または `-v` 指定時に `stderr` へ出力される（調査結果本体は `stdout`）
- `-v` を付けると、進捗ログに加えてデバッグログ（`[log]`）も出力される
- 成功時のクエリ別 `stderr` は既定で非表示（失敗時、または `-v` 指定時のみ表示）
- 複数クエリ実行時は、各クエリの完了通知のみ `stderr` に表示し、調査結果本体は全クエリ完了後に `stdout` へまとめて出力する
- **重要な主張・段落・箇条書きには出典を明記**（全ての文に機械的に付ける必要はない）
- **調査内容の前に「調査理由」を 1〜3 文で記述**
- `調査理由` / `調査結果` / `追加調査` / `出典一覧` の見出しを含める
- `研究・論文収集` の依頼（保存候補抽出）では、`paper-download-kyoto-u` への引き渡しを想定して、候補欄を次の JSON 形式で追加で返却する。

```json
{
  "paper_candidates": [
    {
      "article_title": "論文タイトル（原文・主キー）",
      "title": "論文タイトル（原文・互換キー、将来の互換性用）",
      "title_normalized": "比較用に正規化したタイトル",
      "doi": "10.xxx/xxx（不明な場合は空文字）",
      "article_url": "論文ページURL",
      "pdf_url": "直接PDF URL候補（複数ある場合は最有力1件）",
      "year": 2024,
      "publisher": "出版社/プレプリント",
      "relevance_score": 0.0,
      "reason": "選定理由（短文）"
    }
  ]
}
```

- `relevance_score` は 0.0〜1.0。`reason` は保存候補として妥当な根拠だけを入れる。
- `article_title` を最優先キーとして保存し、`title` は `article_title` と同値に保持。
- 取得できない `doi` / `pdf_url` は空文字。`pdf_url` が不明の候補は「保存不可」と判断して除外。
- `paper_candidates` を上流側から受ける場合は、重複除去のため `article_url` と `doi` を比較して昇順でユニーク化し、同一タイトルを除外する。
- エージェントは、保存前にタイトル整合性・URL解決性・重複を判断し、上流で最終候補を絞ってから `paper-download-kyoto-u` に渡す。

#### 論文収集時の実行例

```bash
bash scripts/web-search.sh -D deep \
  -q "目的: Kyoto University EZproxy 環境で保存できる論文候補を5件見つける。要件: DOI と PDF URL を優先して抽出し、同一タイトル重複を避ける。" \
  -q "目的: 次の論点の論文を比較: 2020年以降、査読有り、実験再現性に言及、DOI有。" \
  -f json
```

この構成では、取得した `paper_candidates` を上流の重複除去後、`paper-download-kyoto-u` の保存実行に渡す。

### 追加の出力要件（研究タスク）
- 出典ごとに種別（一次/二次/ブログ等）を明記し、一次情報を優先すること
- 一次情報が不明な主張は断定しないこと
- **レポートは体系的かつ十分に詳細にするが、重複や言い換えの反復は避けること**

### 検索の進め方

- 関連性の高い情報を抽出し、必要に応じて複数ソースを統合する
- 公式・一次情報（公式ドキュメント、リリースノート、一次発表）を優先する
- 情報が不足する場合は、クエリを具体化して再検索する
- 初回結果で重要な不足点や矛盾が残る場合は、追加調査を行って更新する
- **時間効率のため、1回の実行で複数の検索文（`-q` 複数指定）を与え、目的別に調査すること**
- `-D deep` の既定並列度は `2`。最大安定性が必要な場合は `-j 1`、さらに速度を上げる実験をする場合だけ `-j 3` 以上を検討すること
- **デフォルト検索との分担**:
  - 深掘り・複数ラウンド・出典整理は本スキル（agy）を使用する
  - 簡単なブラウジングや単発の確認はデフォルト検索を使用する
  - 本スキルの結果を見て「追加で詳しく知りたい点」をデフォルト検索で補完し、差分チェックに使う
- 依頼内容がキーワード列の場合は、調査意図が伝わる詳細な指示文に書き換えてから検索する

### Deep Research ベストプラクティス（要約）

- **目的と意思決定を明確化**し、必要な確度を指定する
- **スコープを限定**（期間・地域・製品/バージョン）してノイズを減らす
- **一次情報を優先**し、二次情報は補助・検証用に限定する
- **観点を明示**して比較軸・評価軸を固定する
- **追加調査の差分更新**（不足点→追加検索→結論更新）を必ず行う
